# Plan: Deploy Hugo site to Azure Container Apps via GitHub Actions

## Context

- Hugo static site packaged as Docker image (nginx)
- CI already builds and pushes to `ghcr.io/tn8or/tommyeriksen:latest` on push to `main`
- Self-hosted runner used for build (`runs-on: tommyeriksen`)
- Goal: add a deploy job that pulls the image from GHCR and deploys to Azure Container Apps (free tier)
- Ingress: HTTP only on port 80, SSL handled externally
- Authentication: Azure OIDC (no long-lived secrets)
- Region: `swedencentral` (Gävle) — `swedenwest` does not exist

---

## Phase 1: Azure one-time setup (manual, run once)

Run these commands locally with `az` CLI (already logged in):

### 1.1 Create App Registration + Service Principal with federated credential

```bash
az ad sp create-for-rbac \
  --name "github-tommyeriksen" \
  --role contributor \
  --scopes /subscriptions/<SUBSCRIPTION_ID>/resourceGroups/rg-tommyeriksen \
  --federated-credential-principal github-repository:tn8or/tommyeriksen:ref:refs/heads/main
```

Capture the output. Values needed for GitHub secrets:
- `clientId` → `AZURE_CLIENT_ID`
- `tenantId` → `AZURE_TENANT_ID`
- `subscriptionId` → `AZURE_SUBSCRIPTION_ID`

> **Note:** `--federated-credential-principal` requires Azure CLI v2.36+. The `ref:refs/heads/main` scope restricts OIDC to pushes on `main` only.

### 1.2 Create Resource Group

```bash
az group create --name rg-tommyeriksen --location swedencentral
```

### 1.3 Create Container Apps Environment

```bash
az containerapp env create \
  --name cae-tommyeriksen \
  --resource-group rg-tommyeriksen \
  --location swedencentral
```

### 1.4 Create Container App

```bash
az containerapp create \
  --name ca-tommyeriksen \
  --resource-group rg-tommyeriksen \
  --environment cae-tommyeriksen \
  --image ghcr.io/tn8or/tommyeriksen:latest \
  --registry-server ghcr.io \
  --target-port 80 \
  --ingress external \
  --transport http \
  --min-replicas 0 \
  --max-replicas 1 \
  --cpu 0.25 \
  --memory 0.5Gi
```

After creation, note the FQDN from the output — that's the public URL.

## Phase 2: GitHub Secrets

Add these to the repo (`Settings → Secrets → Actions`):

| Secret | Value |
|---|---|
| `AZURE_CLIENT_ID` | From step 1.1 `clientId` |
| `AZURE_TENANT_ID` | From step 1.1 `tenantId` |
| `AZURE_SUBSCRIPTION_ID` | From step 1.1 `subscriptionId` |

No client secret needed — OIDC uses federated identity.

## Phase 3: Workflow changes

Add a `deploy` job to `.github/workflows/ci.yml`:

```yaml
  deploy:
    needs: build-and-push
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    permissions:
      id-token: write   # required for OIDC
      contents: read
      packages: read    # required for GHCR pull
    steps:
      - name: Azure Login (OIDC)
        uses: azure/login@v2
        with:
          client-id: ${{ secrets.AZURE_CLIENT_ID }}
          tenant-id: ${{ secrets.AZURE_TENANT_ID }}
          subscription-id: ${{ secrets.AZURE_SUBSCRIPTION_ID }}

      - name: Update Container App image
        uses: azure/cli@v2
        with:
          inlineScript: |
            az containerapp update \
              --name ca-tommyeriksen \
              --resource-group rg-tommyeriksen \
              --image ghcr.io/tn8or/tommyeriksen:latest \
              --registry-server ghcr.io \
              --registry-username ${{ github.actor }} \
              --registry-password ${{ secrets.GITHUB_TOKEN }}
```

**Key decisions:**
- `runs-on: ubuntu-latest` — needed for Azure CLI (self-hosted runner may not have `az`)
- `id-token: write` — enables OIDC token exchange with Azure
- `packages: read` — allows the deploy job's `GITHUB_TOKEN` to read GHCR packages
- `GITHUB_TOKEN` for registry auth — ephemeral per job, more secure than a PAT; the image pull happens synchronously during `update` while the token is still valid

## Container App config

- **Image:** `ghcr.io/tn8or/tommyeriksen:latest`
- **Target port:** 80
- **Ingress:** external, HTTP (no TLS)
- **Scale:** 0–1 replicas (scales to zero = free tier eligible when idle)
- **Resources:** 0.25 vCPU, 0.5 GiB memory (free tier: 180k vCPU-sec/mo, 360k GiB-sec/mo)

## Free tier limits reminder

Azure Container Apps free tier gives 180,000 vCPU-seconds and 360,000 GiB-seconds per month. At 0.25 vCPU / 0.5 GiB with scale-to-zero, a personal blog will stay well within limits.
